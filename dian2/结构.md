已完成项目结构分析。这个仓库是一个 **原生 PHP 商城系统（mcy-shop）**，采用 `app(业务层) + assets(前端静态资源) + config(配置) + kernel(底层框架) + runtime(运行时缓存)` 的分层结构，后台（Admin）与用户端（User）基本是“Controller + View + JS 控制器”一一对应。启动链路是 `index.php -> kernel/Kernel.php -> FPM.php/CLI.php`。

下面按“每个文件夹干什么 + 有什么文件”给你完整说明（已按当前仓库实际目录整理）：

## 7) 额外结论（架构特点）
- 典型分层：`Controller -> Service -> Model`，并配合 `Validator`、`Entity`、`Interceptor`。
- 后台 Admin 与用户端 User 的目录设计高度对称。
- 前端资源采用“页面控制器 JS（assets/*/controller）+ 模板（app/View）”配合。
- `kernel/` 是完整自研内核，包含注解、容器、数据库、插件、模板、WAF、会话等基础能力。
