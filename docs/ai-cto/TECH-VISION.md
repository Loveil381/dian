# 技术愿景

最近更新时间：2026-04-02
当前轮次：#18

## 架构评判

项目已经从早期的单文件堆叠状态，演进为适合长期维护的轻量商城架构：

- 前台以 `index.php` 为统一路由入口，页面模板集中在 `templates/`
- 后台完成渐进式 MVC 拆分，形成 `admin/controllers/`、`admin/views/`、`admin/includes/`
- 公共能力沉淀到 `includes/`，包括数据库、CSRF、错误页、邮件、日志、分页、订单状态等
- 前端交互统一收口到 `assets/js/`，通过 `data-*` 与事件委托减少模板内联逻辑
- 样式逐步组件化，形成以 `assets/css/site.css` 和 `assets/css/mobile.css` 为核心的轻量样式体系

当前技术形态已经满足“零框架、低成本部署、可维护、可演进”的目标。

## 已达成的创新点

- 零框架商城：纯 PHP 8.1+、PDO、原生 HTML/CSS/JS，无额外运行时负担
- 安装向导链路：`install.php -> database/schema.sql -> config/installed.lock`
- 双模邮件发送：优先 SMTP socket，失败或未配置时自动回退 `mail()`
- 统一日志工具：`shop_log()` 写入 `logs/app.log`，失败回退 `error_log()`
- 事件委托前端：以 `data-*` 属性承载交互语义，减少模板内联函数
