# 技术决策记录

## ADR-001：零框架原生技术栈

- 日期：2026-04-02
- 状态：已采纳
- 决策：使用原生 PHP + PDO + 原生 HTML/CSS/JS 构建系统
- 原因：适配廉价虚拟主机，降低部署与迁移成本，避免框架引入额外复杂度

## ADR-002：渐进式 MVC 拆分

- 日期：2026-04-02
- 状态：已采纳
- 决策：后台拆分为 `admin/controllers/`、`admin/views/`、`admin/includes/`
- 原因：在最小风险下提升结构清晰度与可维护性

## ADR-003：统一事件委托模式

- 日期：2026-04-02
- 状态：已采纳
- 决策：前台交互统一使用 `data-*` 属性与集中事件委托
- 原因：降低模板内联脚本数量，减少 DOM 与 JS 耦合

## ADR-004：双模邮件发送

- 日期：2026-04-02
- 状态：已采纳
- 决策：优先使用 SMTP socket，失败时回退到 PHP `mail()`
- 原因：兼顾生产可用性与低门槛主机兼容性

## ADR-005：安装向导链路

- 日期：2026-04-02
- 状态：已采纳
- 决策：采用 `install.php -> schema.sql -> installed.lock` 的安装链路
- 原因：统一首次部署流程，减少手工建表和配置出错概率

## ADR-006：统一日志

- 日期：2026-04-02
- 状态：已采纳
- 决策：通过 `shop_log()` 写入 `logs/app.log`，失败时回退 `error_log()`
- 原因：统一日志入口，方便部署后接入采集和监控

## ADR-007：安装向导整合管理员创建

- 日期：2026-04-03
- 状态：已采纳
- 决策：将管理员账号创建整合进 `install.php` 安装向导，消除 `admin/setup.php` 独立入口
- 原因：减少攻击面，统一安装流程，一步完成“建表 + 建管理员”

## ADR-008：价格校验以服务端真实数据为准

- 日期：2026-04-03
- 状态：已采纳
- 决策：购物车与结算流程均从服务端重新获取商品与 SKU 价格，不信任客户端提交的价格字段
- 原因：防止通过表单篡改 `sku_price` 或购物车快照绕过真实价格
