# SPEC — 在线咨询功能（聊天气泡）

> 状态：草案
> 创建：2026-04-05
> 关联：首页 `home-consult-fab` 按钮 + "药师在线小课堂" 区块

---

## 1. 背景与动机

首页右下角已有一个紫色聊天气泡按钮（`chat_bubble` 图标），点击后 `href="#"` 无任何响应。首页底部也有"药师在线小课堂"区块，文案提到"请随时咨询我们的在线药师"。

这两个 UI 元素暗示了一个"在线咨询"能力，但目前没有任何后端实现。需要明确这个功能**做成什么形态**。

## 2. 方案选型

| 方案 | 复杂度 | 依赖 | 适合场景 |
|---|---|---|---|
| A. 跳转外部客服（微信/QQ） | 极低 | 无 | 已有客服渠道，只需引流 |
| B. 弹出联系信息弹窗 | 低 | 无 | 展示微信二维码/电话/邮箱 |
| C. 站内实时聊天 | 高 | WebSocket/轮询 + 聊天记录表 + 后台聊天面板 | 专业客服体验 |
| D. 接入第三方客服（Tidio/Crisp） | 中 | 第三方 JS SDK | 快速上线，功能完整 |

### 推荐：方案 B（联系信息弹窗）

理由：
- 项目铁律禁止引入外部 JS 库（排除 D）
- 站内实时聊天对零框架 PHP 项目过于重量级（排除 C）
- 方案 A 过于简陋
- 方案 B 零依赖、体验好、可后台配置

## 3. 功能描述

### 3.1 前台交互

1. 用户点击聊天气泡按钮
2. 弹出一个模态弹窗 / 底部抽屉（移动端友好）
3. 弹窗内容：
   - 标题："联系我们"或"在线咨询"
   - 药师头像 + 名称（可选）
   - 微信二维码图片（从后台设置读取）
   - 微信号文字（可复制）
   - 手机号 / 电话（可选，可点击拨打）
   - 营业时间 / 回复时间提示
   - 关闭按钮
4. 点击弹窗外部或关闭按钮 → 关闭弹窗

### 3.2 后台配置

在后台"系统设置"tab 中新增"在线咨询"设置区：
- 微信二维码图片（复用现有图片上传）
- 微信号（文本）
- 联系电话（文本）
- 咨询提示语（文本，如"工作日 9:00-18:00 回复"）
- 是否启用咨询按钮（开关）

数据存储在 `{prefix}settings` 表，key 前缀 `consult_`：
- `consult_enabled` — 1/0
- `consult_wechat_qr` — 图片路径
- `consult_wechat_id` — 微信号
- `consult_phone` — 电话
- `consult_notice` — 提示语

### 3.3 显示逻辑

- `consult_enabled = 0` 时，首页不渲染聊天气泡和药师区块
- 微信二维码和微信号至少填一个才显示微信区域
- 电话为空则不显示电话区域

## 4. 涉及文件

| 文件 | 改动 |
|---|---|
| `templates/index.php` | 聊天气泡点击 → 打开弹窗；药师区块读取设置 |
| `assets/js/site.js` | 弹窗打开/关闭逻辑 |
| `assets/css/site.css` | 弹窗样式（对齐 Ethereal 设计体系） |
| `admin/views/settings.php` | 新增"在线咨询"设置区域 |
| `admin/controllers/setting_actions.php` | 保存咨询设置 |
| `admin/data_loaders/settings.php` | 加载咨询设置 |

## 5. 不做的事

- 不做站内实时聊天
- 不做消息记录
- 不引入任何第三方 SDK
- 不做用户端发送消息的能力

## 6. 成功标准

- [ ] 聊天气泡点击后弹出联系信息弹窗
- [ ] 弹窗在移动端（375px）正常展示，无横向滚动
- [ ] 后台可配置微信二维码、微信号、电话、提示语
- [ ] 后台关闭开关后，前台不显示聊天气泡
- [ ] 所有动态输出使用 `shop_e()` 转义
- [ ] CSS 使用 design-tokens 变量，无硬编码
